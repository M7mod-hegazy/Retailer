So: keep, split, or delete the current controls?

A

Split — Designer owns layout, settings tab keeps data (Recommended)

Layout controls (toggles, fonts, margins, custom text) are removed from the old panel and live only in the Designer = one source of truth, no sync bugs. The settings tab keeps tax/currency/prefixes/company info/logo and gains an "Open Designer" button per doc type. Your existing settings auto-seed the first layout, so nothing is lost.

B

Keep both, kept in sync

Old toggles AND the Designer both edit layout, mirrored two-way. Familiar for existing users, but two sources of truth → real risk of drift/bugs (the kind we already found in the audit).

C

Delete the panel entirely — everything in the Designer

Even tax/currency/company info move into the Designer. Cleanest single screen, but those values are shared across all documents, so editing them inside a per-doc designer is awkward and easy to get inconsistent.
